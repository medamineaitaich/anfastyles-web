Change 3: Fix Mobile Horizontal Overflow / Sideways Shifting

1. Root cause
The mobile sideways drift came from a mix of shared and page-level overflow issues:
- root scrolling was only clipped on body
- the mobile header could exceed the viewport
- some mobile rows were forced into single-line layouts
- the Checkout fixed bottom CTA could widen the viewport on smaller phones

2. Files changed
- src/index.css
- src/components/Header.jsx
- src/pages/HomePage.jsx
- src/pages/ShopPage.jsx
- src/pages/CheckoutPage.jsx

3. Exact changes made
- Added horizontal overflow protection to html, body, and #root.
- Disabled horizontal overscroll drift.
- Added a global media safety rule so images, video, and canvas cannot exceed container width.
- Hardened the shared header against mobile overflow with min-w-0 and truncation.
- Changed the Home "Fresh drops" heading row to wrap on mobile.
- Added min-w-0 to the Shop main content column and allowed the toolbar to wrap.
- Made the mobile Checkout sticky bottom bar more resilient and allowed the CTA button to flex.

4. Test result

Change 6: Fix Add-to-Cart Success Message / Toast Timing

1. Root cause
The add-to-cart success feedback used a plain toast.success('Added to cart') call with no stable toast id and no shorter interaction-specific duration. Repeated clicks could create awkward duplicate behavior, and the cart drawer opening immediately afterward could make the toast feel inconsistent because the toast host was not explicitly layered above the drawer overlay.

2. Files changed
- src/pages/ProductDetailPage.jsx
- src/components/ui/sonner.jsx

3. Exact changes made
- Added a stable add-to-cart toast id constant.
- Updated the success toast to reuse that id and use a shorter 2200ms duration.
- Raised the Sonner toaster above the cart drawer/sheet overlay with z-[70].
- This preserved the existing design language while making the confirmation feel more immediate and predictable.

4. Test result
- npm.cmd run build: passed
- No checkout, payment, or API logic was changed.

5. Remaining blockers
- No code blockers.
- A quick manual add-to-cart browser check was still recommended.
